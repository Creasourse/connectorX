# LivePulse Client Connector 部署指南

## 部署概述

本文档提供 LivePulse Client Connector 服务的详细部署步骤。部署需要严格按照顺序执行：

1. **第一步**：启动 client-manager 服务（客户端连接器管理服务）
2. **第二步**：启动 wecom-open 服务（企业微信开放平台连接器服务）

> **注意**：必须先启动 client-manager，再启动 wecom-open，否则服务注册将失败。

## 依赖服务安装

### 1. 安装 Nginx

```bash
# 安装 Nginx
apt install nginx -y

# 将 www-data 加入 cs_conn 用户组
sudo usermod -a -G cs_conn www-data
```

### 2. 安装 PostgreSQL 15

```bash
# 添加 PostgreSQL 官方仓库
sudo sh -c 'echo "deb http://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'

# 导入签名密钥
wget --quiet -O - https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo apt-key add -

# 更新软件包列表
sudo apt update

# 安装 PostgreSQL 15
sudo apt install postgresql-15 postgresql-client-15 -y

# 启动并设置开机自启
sudo systemctl start postgresql@15-main
sudo systemctl enable postgresql@15-main
```

### 3. 初始化数据库

```bash
# 切换至 postgres 用户并进入 psql 终端
sudo -u postgres psql
```

在 psql 终端执行以下 SQL：

```sql
-- 创建用户
CREATE USER cs_conn WITH PASSWORD 'abc123';

-- 创建数据库
CREATE DATABASE cs_client_conn OWNER cs_conn;

-- 授权
GRANT ALL PRIVILEGES ON DATABASE cs_client_conn TO cs_conn;

-- 退出
\q
```

### 4. 安装 Nacos

```bash
# 下载 Nacos 2.5.1
wget https://github.com/alibaba/nacos/releases/download/2.5.1/nacos-server-2.5.1.tar.gz

# 解压
tar -zxvf nacos-server-2.5.1.tar.gz

# 进入 bin 目录
cd nacos/bin

# 单机模式启动
./startup.sh -m standalone
```

### 5. 安装 Redis

```bash
# 安装 Redis
apt install redis-server -y

# 编辑配置文件设置密码
vi /etc/redis/redis.conf
```

在文件末尾添加：

```
requirepass abc123
```

```bash
# 启动 Redis
systemctl start redis

# 设置开机自启
systemctl enable redis-server
```

### 6. 安装 Docker 和 Docker Compose

```bash
# 卸载旧版本
sudo apt remove docker docker-engine docker.io containerd runc -y

# 安装依赖工具
sudo apt update
sudo apt install apt-transport-https ca-certificates curl software-properties-common -y

# 添加 Docker 官方 GPG 密钥
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# 添加 Docker 软件源
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装 Docker
sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io -y

# 安装 Docker Compose
mkdir -p ~/.docker/cli-plugins
curl -SL https://github.com/docker/compose/releases/download/v2.39.3/docker-compose-linux-x86_64 -o ~/.docker/cli-plugins/docker-compose
chmod +x ~/.docker/cli-plugins/docker-compose

# 验证安装
docker compose version
```

## 配置文件准备

### 1. 创建 .env 文件

在项目根目录（与 docker-compose.yml 同目录）创建 `.env` 文件：

```bash
vi .env
```

添加以下内容：

```bash
# Java 应用参数
JAVA_OPTS=-Xms512m -Xmx1024m

# Spring 配置
SPRING_PROFILES_ACTIVE=prod

# Nacos 配置
SPRING_CLOUD_NACOS_SERVER_ADDR=192.168.1.10:8848
SPRING_CLOUD_NACOS_CONFIG_GROUP=DEFAULT_GROUP
SPRING_CLOUD_NACOS_DISCOVERY_GROUP=DEFAULT_GROUP
SPRING_CLOUD_NACOS_CONFIG_NAMESPACE=
SPRING_CLOUD_NACOS_DISCOVERY_NAMESPACE=
SPRING_CLOUD_NACOS_USERNAME=nacos
SPRING_CLOUD_NACOS_PASSWORD=nacos

# 端口配置
APP_PORT_CLIENT_MANAGER=24001
APP_PORT_WECOM_OPEN=23003

# 镜像标签
TAG=latest
```

> **注意**：请将 `SPRING_CLOUD_NACOS_SERVER_ADDR` 修改为实际的 Nacos 服务器地址。

### 2. 确认 docker-compose.yml 文件

确保 `docker-compose.yml` 文件存在于项目根目录，并包含以下服务：

| 服务名 | 容器名 | 端口 | 说明 |
|--------|--------|------|------|
| client-manager | client-manager-connetor | 24001 | 客户端连接器管理服务 |
| wecom-open | wecom-open-connetor | 23003 | 企业微信开放平台连接器服务 |

## 启动 client-manager 服务

### 第一步：启动 client-manager

```bash
# 进入项目目录
cd livepulse-connector-client-manager

# 仅启动 client-manager 服务
docker compose up -d client-manager

# 查看服务状态
docker compose ps

# 查看启动日志
docker compose logs -f client-manager
```

### 第二步：验证 client-manager 启动成功

```bash
# 检查容器运行状态
docker ps | grep client-manager

# 检查健康状态
curl http://localhost:24001/actuator/health

# 在 Nacos 控制台验证服务注册
# 访问：http://192.168.1.10:8848/nacos
# 查看服务列表，确认 client-manager 已注册
```

**成功标志**：
- 容器状态为 `Up`
- 健康检查返回 `{"status":"UP"}`
- Nacos 服务列表中存在 client-manager 服务

## 启动 wecom-open 服务

> **重要**：请确保 client-manager 已成功启动并注册到 Nacos 后，再执行以下步骤。

## Nginx 配置

### 1. 部署 Nginx 配置

```bash
# 复制配置文件到 Nginx 配置目录
cp client_conn_nginx.conf /etc/nginx/sites-available/client-connectorx

# 创建软链接启用站点
ln -s /etc/nginx/sites-available/client-connectorx /etc/nginx/sites-enabled/

# 测试配置
nginx -t

# 重载 Nginx
nginx -s reload
```

### 2. 路由规则说明

| 路径 | 目标服务 | 说明 |
|------|---------|------|
| `/` | 前端静态页面 | Vue 前端应用 |
| `/api/*` | client-manager:24001 | 客户端管理 API |
| `/wecom-open/*` | wecom-open:23003 | 企业微信 API |
| `/images/*` | 图片服务 | 图片代理 |

### 3. 配置 HTTPS（可选）

编辑 `/etc/nginx/sites-available/client-connectorx`，取消 SSL 相关注释并更新证书路径：

```nginx
listen 443 ssl;
ssl_certificate  /home/cs_conn/nginx_conf/ssl/client-connectorx.livepulse.com.cn_bundle.pem;
ssl_certificate_key /home/cs_conn/nginx_conf/ssl/client-connectorx.livepulse.com.cn.key;
```

## 验证部署

### 1. 服务健康检查

```bash
# 检查 client-manager
curl http://localhost:24001/actuator/health

# 检查 wecom-open

```

访问 Nacos 控制台：`http://192.168.1.10:8848/nacos`

在"服务管理"->"服务列表"中确认以下服务已注册：
- client-manager 服务
- wecom-open 服务

### 3. 端口连通性测试

```bash
# 检查端口监听状态
netstat -tulpn | grep -E '24001|23003'

# 预期输出：
# tcp        0      0 0.0.0.0:24001           0.0.0.0:*               LISTEN      [docker-proxy]
# tcp        0      0 0.0.0.0:23003           0.0.0.0:*               LISTEN      [docker-proxy]
```

### 4. 通过 Nginx 访问

```bash
# 通过域名访问前端页面
curl http://client-connectorx.livepulse.com.cn/

# 通过 Nginx 代理访问 API
curl http://client-connectorx.livepulse.com.cn/api/actuator/health
curl http://client-connectorx.livepulse.com.cn/wecom-open/actuator/health
```

## 常见问题

### Q1: client-manager 启动失败

**可能原因**：
- Nacos 连接失败
- 数据库连接失败
- Redis 连接失败

**排查方法**：
```bash
# 查看容器日志
docker compose logs client-manager

# 检查 Nacos 连接
curl http://192.168.1.10:8848/nacos/v1/console/health/readiness

# 检查数据库连接
psql -h localhost -U cs_conn -d cs_client_conn

# 检查 Redis 连接
redis-cli -a abc123 ping
```

### Q2: wecom-open 无法注册到 Nacos

**可能原因**：
- client-manager 未启动
- Nacos 连接配置错误

**排查方法**：
```bash
# 确认 client-manager 已启动
docker compose ps

# 查看 wecom-open 日志
docker compose logs wecom-open

# 检查 Nacos 服务列表
curl http://192.168.1.10:8848/nacos/v1/ns/instance/list?serviceName=wecom-open
```

### Q3: 容器间网络不通

**排查方法**：
```bash
# 检查 Docker 网络
docker network ls
docker network inspect livepulse-connector-client-manager_app-network

# 从一个容器访问另一个容器
docker exec -it client-manager-connetor curl http://wecom-open-connetor:23003/actuator/health
```
