## 项目概述

**企业微信连接器（livepulse-connector-wecom-open）** 是一个基于 Spring Boot 3.x 开发的企业微信数据同步服务，提供与企业微信 API 的对接能力，支持：

- 企业微信通讯录同步（部门、员工）
- 外部联系人管理
- 客户群管理
- 标签管理
- CDP 标签映射
- 回调事件处理
- 定时调度任务

- **JDK**: 17+

- **PostgreSQL**: 12+

## 配置说明

### 1. Nacos 配置

项目使用 Nacos 作为配置中心和服务注册中心，需要在 `bootstrap.yml` 中配置：

```yaml
spring:
  cloud:
    nacos:
      discovery:
        server-addr: ${NACOS_HOST:localhost}:${NACOS_PORT:8848}
        namespace: ${NACOS_NAMESPACE:cs}
        username: ${NACOS_USERNAME:nacos}
        password: ${NACOS_PASSWORD:nacos}
        group: cs
      config:
        server-addr: ${NACOS_HOST:localhost}:${NACOS_PORT:8848}
        namespace: ${NACOS_NAMESPACE:cs}
        group: cs
        file-extension: properties
```

### 2. 回调配置

在企业微信后台配置回调 URL 时需要以下配置：

```yaml
callback:
  token: LxCIyJL              # 企业微信回调 Token
  encodeAesKey: 3jzqZrQGaIXnQFTBQgeVzrGHhFkIMWipeKhnQ5XJPGT  # 回调加密密钥
```

### 3. 调度任务配置

```yaml
wecom:
  scheduled:
    enabled: true  # 是否启用调度任务

    # 部门+员工信息同步
    department-and-employee:
      enabled: true
      cron: "0 0 2 * * ?"  # 每天凌晨2点

    # 外部联系人同步
    external-contact:
      enabled: true
      cron: "0 0 3 * * ?"  # 每天凌晨3点

    # 客户群同步
    customer-group:
      enabled: true
      cron: "0 0 4 * * ?"  # 每天凌晨4点

    # 企业标签库同步
    corp-tag:
      enabled: true
      cron: "0 0 5 * * ?"  # 每天凌晨5点
```

### 4. 环境变量

| 变量名 | 说明 | 默认值     |
|--------|------|---------|
| `PROFILES_ACTIVE` | 运行环境 | `dev`   |
| `NACOS_HOST` | Nacos 地址 | `localhost`  |
| `NACOS_PORT` | Nacos 端口 | `8848`  |
| `NACOS_NAMESPACE` | Nacos 命名空间 | `cs`    |
| `NACOS_USERNAME` | Nacos 用户名 | `nacos` |
| `NACOS_PASSWORD` | Nacos 密码 | -       |

## 编译打包

### Maven 打包

```bash

cd livepulse-connector-wecom-open

mvn clean package -DskipTests
```


## Docker 部署

### 1. 构建 Docker 镜像

```bash
# 方式一：使用 Maven 构建后打包
cd livepulse-connector-wecom-open
mvn clean package -DskipTests
docker build -t livepulse-connector-wecom-open:latest .

# 方式二：指定版本号
docker build -t livepulse-connector-wecom-open:1.0.0 .
```

### 2. Docker 运行

```bash
# 基础运行
docker run -d \
  --name wecom-open-connector \
  -p 23003:23003 \
  livepulse-connector-wecom-open:latest

# 带环境变量运行
docker run -d \
  --name wecom-open-connector \
  -p 23003:23003 \
  -e SPRING_PROFILES_ACTIVE=prod \
  -e NACOS_HOST=your-nacos-host \
  -e NACOS_PORT=8848 \
  -e NACOS_NAMESPACE=cs \
  -e NACOS_USERNAME=nacos \
  -e NACOS_PASSWORD=your-password \
  livepulse-connector-wecom-open:latest

# 挂载日志目录
docker run -d \
  --name wecom-open-connector \
  -p 23003:23003 \
  -v /data/logs/wecom:/data/logs \
  -e SPRING_PROFILES_ACTIVE=prod \
  livepulse-connector-wecom-open:latest

# 设置 JVM 参数
docker run -d \
  --name wecom-open-connector \
  -p 23003:23003 \
  -e JAVA_OPTS="-Xms1g -Xmx2g" \
  livepulse-connector-wecom-open:latest
```

### 3. Docker Compose 部署

创建 `docker-compose.yml` 文件：

```yaml
version: '3.8'

services:
  wecom-open-connector:
    image: livepulse-connector-wecom-open:latest
    container_name: wecom-open-connector
    restart: always
    ports:
      - "23003:23003"
    environment:
      - SPRING_PROFILES_ACTIVE=prod
      - NACOS_HOST=nacos.livepuls.com.cn
      - NACOS_PORT=8848
      - NACOS_NAMESPACE=cs
      - NACOS_USERNAME=nacos
      - NACOS_PASSWORD=your-password
      - JAVA_OPTS=-Xms1g -Xmx2g
    volumes:
      - ./logs:/data/logs
    networks:
      - livepulse-network
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:23003/actuator/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 60s

networks:
  livepulse-network:
    external: true
```

启动服务：

```bash
# 启动
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止
docker-compose down

# 重启
docker-compose restart
```

### 4. 容器管理命令

```bash
# 查看运行状态
docker ps

# 查看日志
docker logs -f wecom-open-connector

# 进入容器
docker exec -it wecom-open-connector bash

# 重启容器
docker restart wecom-open-connector

# 删除容器

# 查看资源使用情况

```

## API 接口文档

服务启动后，访问 Swagger UI 查看完整 API 文档：

```
http://localhost:23003/swagger-ui.html
```

### 主要接口分类

#### 1. 企业微信账户管理 (`/wecomCorp`)

| 接口 | 方法 | 说明 |
|------|------|------|
| `/wecomCorp/pageList` | POST | 分页查询企业微信账户列表 |
| `/wecomCorp/detail/{id}` | GET | 根据ID查询企业微信账户详情 |
| `/wecomCorp/saveOrUpdate` | POST | 保存或更新企业微信账户 |
| `/wecomCorp/delete/{id}` | DELETE | 删除企业微信账户 |

#### 2. 企业微信数据同步 (`/wecomSync`)

| 接口 | 方法 | 说明 |
|------|------|------|
| `/wecomSync/syncDepartments/{corpId}` | POST | 同步部门列表 |
| `/wecomSync/syncEmployees/{corpId}` | POST | 同步员工列表 |
| `/wecomSync/syncEmployeeDetails/{corpId}` | POST | 同步员工详细信息 |
| `/wecomSync/syncAll/{corpId}` | POST | 全量同步（部门+员工+员工详情） |
| `/wecomSync/syncExternalContacts/{corpId}` | POST | 同步外部联系人列表 |
| `/wecomSync/syncAllExternalContacts/{corpId}` | POST | 全量同步外部联系人 |
| `/wecomSync/syncCustomerGroups/{corpId}` | POST | 同步客户群列表 |
| `/wecomSync/syncAllCustomerGroups/{corpId}` | POST | 全量同步客户群 |
| `/wecomSync/syncCorpTags/{corpId}` | POST | 同步企业标签库 |

#### 3. CDP 标签管理 (`/wecomCdpTag`)

| 接口 | 方法 | 说明 |
|------|------|------|
| `/wecomCdpTag/pageList` | POST | 分页查询 CDP 标签映射 |
| `/wecomCdpTag/detail/{id}` | GET | 查询 CDP 标签映射详情 |
| `/wecomCdpTag/saveOrUpdate` | POST | 保存或更新 CDP 标签映射 |
| `/wecomCdpTag/delete/{id}` | DELETE | 删除 CDP 标签映射 |
| `/wecomCdpTag/sync/{id}` | POST | 同步 CDP 标签到企业微信 |

#### 4. 回调管理 (`/wecomCallback`)

| 接口 | 方法 | 说明 |
|------|------|------|
| `/wecomCallback/callback` | GET/POST | 企业微信回调接口 |
| `/wecomCallback/verify` | GET | 验证回调 URL |
| `/wecomCallback/test` | POST | 手动触发回调测试 |

#### 5. 回调配置管理 (`/wecomCallbackConfig`)

| 接口 | 方法 | 说明 |
|------|------|------|
| `/wecomCallbackConfig/pageList` | POST | 分页查询回调配置 |
| `/wecomCallbackConfig/detail/{id}` | GET | 查询回调配置详情 |
| `/wecomCallbackConfig/saveOrUpdate` | POST | 保存或更新回调配置 |
| `/wecomCallbackConfig/delete/{id}` | DELETE | 删除回调配置 |

#### 6. 元数据管理 (`/metaTable`)

| 接口 | 方法 | 说明 |
|------|------|------|
| `/metaTable/pageList` | POST | 分页查询元数据表 |
| `/metaTable/detail/{id}` | GET | 查询元数据表详情 |
| `/metaTable/saveOrUpdate` | POST | 保存或更新元数据表 |
| `/metaTable/delete/{id}` | DELETE | 删除元数据表 |

## 调度任务配置

调度任务支持 Cron 表达式和固定延迟两种配置方式。

### Cron 表达式示例

| Cron 表达式 | 说明 |
|-------------|------|
| `0 0 2 * * ?` | 每天凌晨 2:00 |
| `0 0 */6 * * ?` | 每 6 小时执行一次 |
| `0 0 0 * * MON` | 每周一凌晨 00:00 |
| `0 30 8 * * MON-FRI` | 工作日早上 8:30 |
| `0 0 0 1 * ?` | 每月 1 号凌晨 00:00 |

### 调度任务状态检查

```bash
# 查看应用日志
tail -f logs/connector-wecom-open/debug.log | grep "调度任务"

# 应该看到类似的日志输出
# ========== 开始执行部门+员工信息同步调度任务 ==========
# 查询到 N 个企业微信账户
# 开始同步企业微信账户 [1] 的部门和员工信息
# ========== 部门+员工信息同步调度任务执行完成 ==========
```

## 常见问题

### 1. 启动失败

**问题**: 服务启动时报错 `Failed to connect to Nacos`

**解决方案**:
- 检查 Nacos 服务是否正常运行
- 确认 `bootstrap.yml` 中 Nacos 配置是否正确
- 检查网络连接和防火墙设置

### 2. 接口返回 404

**问题**: 访问 API 接口返回 404

**解决方案**:
- 确认服务是否正常启动
- 检查接口路径是否正确
- 查看 Swagger 文档确认正确的接口地址

### 3. 数据库连接失败

**问题**: 日志显示数据库连接错误

**解决方案**:
- 检查 Nacos 配置中心的数据库配置
- 确认数据库服务是否正常运行
- 检查数据库连接信息（地址、端口、用户名、密码）

### 4. Docker 容器启动后立即退出

**问题**: Docker 容器启动后状态为 `Exited`

**解决方案**:
```bash
# 查看容器日志
docker logs wecom-open-connector

# 常见原因：
# 1. Nacos 连接失败 - 检查环境变量配置
# 2. 端口冲突 - 检查端口是否被占用
# 3. 内存不足 - 调整 JVM 参数
```

### 5. 调度任务不执行

**问题**: 配置的调度任务没有执行

**解决方案**:
- 检查配置文件中 `wecom.scheduled.enabled` 是否为 `true`
- 确认对应任务的 `enabled` 配置
- 验证 Cron 表达式格式是否正确
- 查看应用日志确认调度任务是否加载

### 6. 企业微信回调验证失败

**问题**: 配置企业微信回调 URL 时验证失败

**解决方案**:
- 确认回调 URL 可以从外网访问
- 检查 `callback.token` 和 `callback.encodeAesKey` 配置是否与企业微信后台一致
- 查看服务日志中的回调请求记录
